# 수납 관리 — 3탭 조회 + 미납자 문자 발송 (설계 문서)

작성일: 2026-06-17

## 배경

`수납 관리`(`/payments`) 화면은 `창구수납`+`수납 내역`+`매출현황`을 통합한 화면이다. 현재는 상단에 금액 카드 3개(완납/환불/미수금) + 도넛 차트(수납 완료율)로 구성된 **대시보드형 요약 블록**과, 그 아래 상태 드롭다운 필터로 전환되는 단일 목록 + 행 클릭 시 마스터-디테일(원생 상세·수납 입력) 구조를 갖는다.

이번 작업은 이 화면을 **조회 중심 3탭 + 미납자 문자 발송** 구조로 재구성한다. KPI·차트 등 대시보드 성격의 시각화는 **추후 별도 "대시보드" 메뉴**로 분리할 예정이므로 이 화면에서는 제거한다.

## 목표

1. 상태별 **3개 탭**으로 조회: 미납 현황 / 완납 현황(환불 포함) / 청구·예정 현황.
2. 각 탭에서 기존 필터(월·이름·그룹·반)로 **조회** 가능.
3. **미납자에게 문자 발송**: 개별(행 단위) + 일괄(다중 선택).
4. 행 클릭 시 기존 **마스터-디테일(원생 상세 + 수납 입력 모달 + 결제 문자 모달)** 흐름은 그대로 유지.

## 비목표 (이번 작업 범위 밖)

- KPI 지표·게이지·추이 차트 등 대시보드 시각화 (추후 별도 메뉴로 분리).
- 실제 문자/알림톡 발송 연동 (기존과 동일하게 미리보기/alert 데모).
- 연체 자동 분류 버킷(수납대기/연체주의/장기미납) 필터 — 데이터 한계로 이번엔 제외(아래 "데이터 한계" 참고).

## 레이아웃

```
수납 관리
판교 캠퍼스 · 조건별 수납 조회 · 미납자 문자 발송
──────────────────────────────────────────────
[ 미납 현황 (14) ] [ 완납 현황 (62) ] [ 청구·예정 (2) ]   ← 탭바
──────────────────────────────────────────────
요약줄:  14건 · 미납 합계 8,400,000원        [선택 3명 일괄 문자]
필터바:  [월] [학생이름] [그룹] [반]  (완납 탭만: [결제수단] [수납일 from~to])
──────────────────────────────────────────────
목록 (탭별 컬럼)  ──행 클릭──▶  기존 마스터-디테일(상세 + 수납입력 모달)
```

- 상단 금액 카드 3개 + 도넛 블록은 **제거**.
- 탭은 기존 "상태 드롭다운"을 대체하므로 필터바에서 상태 셀렉트는 제거한다.
- 마스터-디테일 진입(행 클릭) 시 좌측 축소 목록 + 우측 상세는 현행 유지.

## 탭 = 상태 매핑

파생 상태(`deriveViewStatus`)는 현행대로 `완납 / 미납 / 예정 / 환불`을 사용한다.

| 탭 | 포함 상태 | 목록 컬럼 |
|---|---|---|
| **미납 현황** | `미납` | ☑ · 원생(+상태뱃지) · 반(schedule) · 부모HP(모) · **연체일수** · 미납액 · `[문자]` |
| **완납 현황** | `완납` + `환불` | 원생(+학년) · 반 · 상태(완납/환불 뱃지) · 수단 · 금액 · 수납일 |
| **청구·예정** | `예정` | 원생 · 반 · 부모HP(모) · 납기일 · 청구액 |

- 환불은 완납 탭 안에서 빨강 음수 금액 + `환불` 뱃지로 시각 구분(별도 탭/필터 없음).
- 탭 라벨 옆 괄호 안에 건수 표기(예: `미납 현황 (14)`). 건수는 **필터 적용 전** 월 전체 기준.

## 요약줄(탭별)

각 탭 목록 위 한 줄. 현재 필터가 적용된 `rows` 기준 집계.

- 미납: `N건 · 미납 합계 X원` + 우측 `일괄 문자(선택 M)` 버튼.
- 완납: `N건 · 수납 합계 X원` (환불액 있으면 `(환불 Y원 포함)` 병기).
- 예정: `N건 · 예정 합계 X원`.

## 문자 발송 (미납 탭 전용)

- **일괄**: 미납 행 체크박스로 다중 선택 → 요약줄의 `일괄 문자(M)` 버튼 → 발송 미리보기(alert). 선택 0명이면 비활성.
- **개별**: 각 미납 행의 `[문자]` 버튼 → 기존 **결제 문자 모달**(`showMsgModal`)을 해당 학생 대상으로 열어 발송.
- 실제 발송 연동 없음 → 기존 다른 액션과 동일하게 미리보기/alert 데모.

## 컴포넌트 / 코드 구조

### `app/(admin)/payments/page.tsx` (오케스트레이터)
- 상태 추가: `activeTabKey: '미납' | '완납' | '예정'` (기존 `statusFilter` 셀렉트 대체).
- 상단 금액 카드 + 도넛 블록 제거.
- 탭바 렌더링(건수 포함) + 탭 전환 시 선택(`selectedIds`) 초기화.
- 필터바: 월·이름·그룹·반 공통, 완납 탭에서만 결제수단·수납일 범위 노출.
- 요약줄 렌더링.
- 목록은 `PaymentList`에 `tab` prop 전달.
- 개별 문자: 행의 `[문자]` 클릭 시 해당 학생을 대상으로 `showMsgModal` 오픈(현 모달은 `selectedStudent` 기준이므로, 문자 대상 학생 상태를 모달에 전달하는 방식으로 분리).
- 마스터-디테일·수납 입력 모달·결제 문자 모달 로직 유지.

### `lib/payments.ts`
- `daysOverdue(inv, today): number` — `today`와 `due_date` 차이(미납 행의 연체일수). 음수면 0.
- `rowsForTab(rows, tab): PaymentRow[]` — 탭별 상태 필터(미납 / 완납+환불 / 예정).
- 탭별 요약 계산 헬퍼(건수·합계). 기존 `computeSummary`를 재사용하거나 탭 단위 합계 함수 추가.
- 기존 `filterRows`는 그대로 사용하되, 상태 필터 인자는 탭에서 주입.

### `components/payments/PaymentList.tsx`
- `mode: 'paid'|'unpaid'` → `tab: '미납'|'완납'|'예정'`로 변경, 탭별 컬럼 세트 분기.
- 미납 탭: 체크박스 + `연체일수` 컬럼 + 행 우측 `[문자]` 액션 버튼(`onSendMessage(row)` 콜백) 추가.
- 완납 탭: 현행 paid 컬럼 유지.
- 예정 탭: 납기일·청구액 컬럼.
- 빈 상태 문구는 탭에 맞게(예: "조건에 맞는 미납 원생이 없습니다").

## 데이터 한계

- mock 데이터는 `billing_month: '2026-06'` 단일 월이며, 미납 청구의 `due_date`가 대부분 `2026-06-01`(예정 2건만 `2026-06-30`)이다. 기준일 `TODAY='2026-06-16'` 기준 미납 연체일수가 전부 ~15일로 동일하다.
- 따라서 "연체주의(1~7일)/장기미납(7일+)" 자동 분류 버킷은 데모 변별력이 낮아 이번 범위에서 제외한다. 대신 **연체일수 컬럼**만 표기한다.
- 추후 데모 변별이 필요하면 미납 청구의 `due_date`를 흩뿌려(예: 6/3, 6/10, 6/14 등) 버킷 필터를 도입할 수 있다.

## 테스트 / 검증

- 탭 전환 시 목록·컬럼·요약줄이 상태에 맞게 바뀐다.
- 미납 탭에서 체크 선택 → `일괄 문자(M)` 카운트 반영, 0명 시 비활성.
- 미납 행 `[문자]` → 해당 학생 결제 문자 모달 오픈.
- 행 클릭 → 기존 마스터-디테일/수납 입력 모달 정상 동작.
- 완납 탭 환불 행이 음수·`환불` 뱃지로 구분 표기된다.
- 필터(월·이름·그룹·반, 완납 탭 결제수단·수납일)가 각 탭에서 정상 동작한다.
