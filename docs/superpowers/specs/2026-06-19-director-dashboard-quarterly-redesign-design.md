# 원장(랩장) 대시보드 분기 개편 설계

작성일: 2026-06-19
대상 화면: `app/(admin)/dashboard/page.tsx` 의 `role === '원장'` 분기
선행 문서: `2026-06-19-director-dashboard-redesign-design.md`(현재 구현 상태) 를 잇는 2차 개편.

## 0. 기본 원칙 (Toss 스타일 미니멀)

정보·글자를 최소화해 랩장이 처음 봐도 "쉽다·금방 이해된다·나도 쓰겠다"는 인상을 주는 것이 핵심
목표. Toss 앱처럼 직관적이고 필요한 것만 담는다.

- 메뉴 진입 후 같은 제목 반복, 불필요한 설명 텍스트, 중복 레이블은 제거 대상.
- 디자인 가이드(컬러·타이포)는 전체 빌드 완료 후 일괄 적용. 이 개편에서는 **구조와 텍스트 다이어트**에 집중.

## 1. 새 화면 구성 (위 → 아래)

```
① 인사 배너            유지·간소화 (중복 설명 제거)
② 성장 추이 그래프      신규·최상단 — 학생 수 · 매출, 5개 분기 라인
③ 손익 요약            분기 단위 — 총매출 · 지출 · 영업이익 + 분기 경과%
④ 운영 인사이트(선행)   홍보·상담·신규등록·퇴원율 4개 + 전환율 흐름
⑤ 관리 항목            상담 미결 · 퇴원 가능성 · 납부 현황
```

## 2. 데이터 전략

분기 단위 × 5개 분기(현재 → 전년 동일 분기) 히스토리가 기존 mock에 없으므로 **데모용 분기 데이터
레이어를 신규 생성**한다. 기존 자산은 건드리지 않는다.

- 신규: `lib/quarterly.ts` — 5개 분기(`2025-Q2` ~ `2026-Q2`) 데모 데이터 + 분기 헬퍼.
- 미변경: `lib/pnl.ts`(월 단위) · `/revenue` 페이지 · `lib/lab-metrics.ts`(주 단위, `/ai` 메뉴가 사용).
- 현재 분기(`2026-Q2`)는 **진행 중**이라 매출·지출은 to-date(경과분)로 표기하고 경과% 배지로 맥락을 준다.
  학생 수는 스냅샷이라 분기말 기준으로 깔끔히 증가.

### 데이터 형태

```ts
interface QuarterPnl  { quarter; revenue; expense; profit; students; inProgress }
interface QuarterLead { quarter; promo; consult; newEnroll; withdrawRate; inProgress }
// 타 캠퍼스 비교(선행지표 4개): 분기별 6개 랩 분포 → { min, max, avg, mine }
```

### 분기 헬퍼

`quarterOf(dateISO)`, `quarterLabel`, `quarterShort`, `quarterRange`, `quarterElapsedPct(asOf)`,
`last5Quarters()`, `distributionForQuarter(quarter, key)`.

기준일은 앱 공통 `TODAY`(`2026-06-14`)에 앵커. `Date.now()` 미사용(결정적 테스트).

## 3. Phase별 작업

### Phase 1 — 분기 데이터 레이어 (`lib/quarterly.ts` + `lib/quarterly.test.ts`)

- 내 랩(`lab-pg`, 판교랩) 5분기 손익/학생수/선행지표 = 손으로 작성한 성장 시리즈.
- 타 랩은 결정적 생성(랜덤 금지, `lab-metrics.ts`의 `genLabMetric` 패턴)으로 분포(min/max/avg) 산출.
- 순수 분기 수학 헬퍼는 vitest로 검증.

### Phase 2 — 성장 추이 + 손익 (②③)

- **② `QuarterlyTrends`(신규)**: 학생 수·매출 두 개의 5분기 라인(또는 막대) 차트. 잔액 제외(은행 API 불가).
  진행 중 분기 포인트는 흐리게 + "진행 중" 표시.
- **③ `PnlSummaryStrip`(분기화)**: 핵심 3카드 **총매출 · 지출 · 영업이익**. `총지출`→`지출` 표기,
  "가맹운영세트" 등 중복 텍스트 삭제. 상단에 `2026 2분기 · NN% 경과` 배지.
  **면세·과세 매출 보조 2줄 제거**(미니멀 원칙). `/revenue` 링크는 유지.

### Phase 3 — 운영 인사이트(선행지표) 재편 (④)

- `AiBenchmarkHero` → 8지표에서 **4지표(홍보 건수·상담 건수·신규 등록·퇴원율)**로 축소.
  출석률·보강 완료율·재등록·수납률 제거.
- **디폴트 = 현재 분기**, "5개 분기 비교" 토글로 펼침.
- 각 지표 막대에 **min · max · 평균 + 자사 수치** 표시(타 캠퍼스 평균 비교 유지).
- **전체 순위 표시 제거**(데모). **AI 인사이트 문구 제거**(텍스트 최소화).
- 하단 전환율 흐름(홍보→상담→등록)은 기존 `ConsultFunnelCard` 유지.

### Phase 4 — 관리 항목 (⑤)

- **상담 미결**(`ConsultGapList` → leads 기반 교체): 문의 후 등록/미등록 미확정자(`신규문의·상담예약·
  상담완료`) 목록. 라벨 "미정"→"미결", `lib/leads.ts`의 `activeLeads` 사용.
- **퇴원 가능성 학생**(`AtRiskList` 유지 + 보강): 리스트 유지하고 상단에 판정 기준 한 줄
  (출석 저조·미납·휴원 기반) 병기.
- **납부 현황**: 현행 유지.
- 인사 배너 등 화면 전반의 중복/불필요 텍스트 다이어트.

## 4. 비범위 (Out of scope)

- 실제 분기 집계 연동(데모 mock 사용), 은행 잔액 API, 전체 순위, 컬러/타이포 디자인 가이드 일괄 적용.
- 교사·SO 대시보드(`role !== '원장'`) 및 `/ai`·`/revenue` 페이지는 변경하지 않는다.
