# 수납 관리 페이지 보강 — 설계 문서

- 작성일: 2026-06-16
- 대상 파일: `app/(admin)/payments/page.tsx` (단일 페이지), 보조로 `lib/mock-data.ts`
- 전제: mock-data 기반 프론트 프로토타입 (실제 백엔드 연동 아님)

## 1. 목표

레거시 3개 화면(창구수납 · 조건별 수납 조회 · 미납/예정자 조회)의 유용한 기능을
현재 `/payments` 한 페이지 안에 흡수한다.

핵심 원칙: **디지털 친숙도가 낮은 가맹점주도 헤매지 않도록, 탭 전환 없이 한 곳에서 다 본다.**

- 상단 뷰 전환 탭 금지.
- "조건별 수납 조회"를 메인으로 두고, 미납/예정자 조회를 같은 화면에서 처리.
- 개별 학생 기능은 목록 행 클릭 → 우측 상세 패널 (원생 상세보기와 동일 성격, 중복 기능은 늘리지 않음).

## 2. 채택안 (A: 요약 카드를 그대로 필터로)

브레인스토밍에서 A안 확정. 상태 카드를 누르면 아래 목록이 그 그룹으로 걸러지고,
선택한 카드에 따라 목록이 **두 모드 중 하나**로 자동 전환된다. 새 컨트롤 학습 비용 0, 탭 0.

## 3. 화면 구성

### 3.1 상태 카드 줄 (클릭 = 필터, "사람" 축 하나로 통일)

한 줄에 5개 카드. 클릭하면 `statusFilter`가 바뀌고 목록·합계 띠가 갱신된다.

| 카드 | 의미 | 선택 시 목록 모드 |
|------|------|------------------|
| 전체 | 해당 월 전체 인원 | 결제 내역 모드 |
| 완납 | status === '완납' | 결제 내역 모드 |
| 미납 | status === '미납' | 연락·통지 모드 |
| 예정 | 납기 도래 예정 (미납 + due_date 미래) | 연락·통지 모드 |
| 환불 | status === '환불' | 결제 내역 모드 (금액 음수) |

각 카드는 건수(명)를 크게 표시. 선택된 카드는 강조(완납=초록/주황 기조, 미납=빨강, 예정=주황, 환불=보라).

### 3.2 돈 합계 띠 (읽기전용, 클릭 축과 분리)

카드 줄 아래 한 줄. 현재 필터 결과 기준 금액을 보여준다.

- 기본/완납/전체: `총수납 · 카드 · 현금·계좌 · 환불` 분리 합계.
- 미납/예정 선택 시: `미납(예정) 합계 · N명`.

### 3.3 보조 도구줄

- 월 선택(`type="month"`, 기본 2026-06), 이름·반 검색 인풋.
- 우측: 결제 내역 모드 → `엑셀` 버튼(미리보기/안내 수준). 연락·통지 모드 → `일괄 문자`, `통지서 인쇄` 버튼.

### 3.4 목록 — 두 모드

**(a) 결제 내역 모드** (전체/완납/환불)
컬럼: `원생(이름+학년) · 반 · 상태(배지) · 수단 · 금액(우측, 환불은 음수/빨강) · 수납일`.
현재 페이지 현황 테이블 구조 유지하되 환불 표시 추가.

**(b) 연락·통지 모드** (미납/예정)
컬럼: `체크박스 · 원생(이름+상태배지) · 반 · 부모HP(모) · 미납액(우측)`.
- 체크박스로 다중 선택 → 상단 `일괄 문자` / `통지서 인쇄` 대상.
- 예정 행은 '예정' 배지(주황), 미납 행은 '미납' 배지(빨강).

두 모드 모두 행 클릭 → 우측 상세 패널 진입(마스터-디테일).

### 3.5 상세 패널 (개별 학생)

행 클릭 시 현재의 마스터-디테일 상세를 사용한다. 보강 항목:

- 정보줄에 (있으면) `가상계좌`, `현금영수증No`, `장학 정보`, `재원 형제` 칩 추가.
- 액션 버튼에 `납입증명서 인쇄`(미리보기/안내 수준) 추가. 기존 `수납 입력`, `결제 문자` 유지.
- 청구 세부내역 / 수납 이력 탭(전체·미납·수강료·교재·기타) = 기존 영역 그대로 유지.

개별 수납이력은 원생 상세보기와 동일 성격이므로 여기서는 가볍게(요약+이력+입력) 유지하고
중복 기능을 늘리지 않는다.

## 4. 데이터 모델 보강 (`lib/mock-data.ts`)

기존 구조를 최대한 활용하고, 부족한 것만 옵셔널로 추가한다.

- **예정(InvoiceStatus)**: 일부 invoice를 `due_date` 미래로 두고 미납 상태로 생성해 "예정"을 파생.
  (별도 status 값 추가 대신 `status==='미납' && due_date > 오늘` = 예정으로 계산. 단순 우선.)
- **환불 표본**: 최소 1~2건 `status==='환불'` + 음수 금액 payment 추가(레거시 연제현 사례 모사).
- **수납자(`Payment.collector?`)**: 원장님/리암/키오스크/온라인결제 등. (목록에는 기본 노출 안 함, 추후 확장 여지)
- **학생 보강 표시용**: `virtual_account?`, `scholarship_type?`를 Student(또는 파생)에 옵셔널 추가.
  `sibling_ids`는 이미 존재 → 형제 칩에 사용. `cash_receipt_no`는 이미 Payment에 존재.

> 모든 신규 필드는 옵셔널이며, 없으면 칩/컬럼을 자연스럽게 생략한다(빈 값 노출 금지).

## 5. 상태(state) 설계

기존 useState 유지 + 다음을 조정:

- `statusFilter`: '전체' | '완납' | '미납' | '예정' | '환불' (카드 클릭으로 설정).
- `listMode` = statusFilter로부터 파생 (`미납`/`예정` → 연락·통지, 그 외 → 결제 내역).
- `selectedIds: Set<string>` — 연락·통지 모드 다중 선택.
- 기존 `selectedStudent`, 모달 상태(`showPayModal`, `showMsgModal`) 유지.

`useMemo`로 월·검색·상태 필터링 후 행 산출, 합계 띠 값 파생.

## 6. 범위에서 제외 (YAGNI)

- 상단 뷰 전환 탭 / 별도 라우트(`/payments/report` 등) — 명시적 제외.
- 레거시의 과다 필터(강사명·담임명·유입경로·디랩을선 등) — 이름·반 검색으로 충분.
- 카드수수료 제외, 청구자료 일괄 생성, 수강증출력 토글 — 1차 범위 밖(추후 별도 스펙).
- 실제 인쇄/엑셀/문자 전송 연동 — 미리보기·안내 수준(프로토타입).

## 7. 컴포넌트 경계

`payments/page.tsx` 한 파일이 커지므로, 읽기 쉽게 내부를 분리한다(같은 폴더 하위 파일 권장):

- `StatusCards` — 상태 카드 줄 + 합계 띠 (props: 집계값, 선택상태, onSelect).
- `PaymentListPaid` — 결제 내역 모드 목록.
- `PaymentListUnpaid` — 연락·통지 모드 목록(체크박스/일괄 액션 대상 선택).
- `StudentDetailPanel` — 우측 상세(요약 칩 + 청구 + 이력 탭 + 액션).
- 모달(`PayModal`, `MsgModal`)은 기존 마크업을 컴포넌트로 추출.

각 단위는 props 인터페이스만으로 동작/테스트 가능해야 한다.

## 8. 검증 기준

- 카드 클릭 시 목록·합계 띠가 해당 그룹으로 즉시 전환된다.
- 미납/예정 카드 선택 시 목록이 연락·통지 모드(체크박스+연락처)로 바뀌고 일괄 액션 버튼이 노출된다.
- 환불 건이 음수 금액으로 목록과 합계에 반영된다.
- 행 클릭 시 상세 패널이 열리고, 보강 칩/버튼이 값 있을 때만 표시된다.
- 빌드/타입 통과(`next build` 또는 타입체크).
