# 강의실별 배치표 — 원생 빠른 패널 (미납 관리) 설계

작성일: 2026-06-18
대상 화면: 시간표 → 강의실별 배치 (`PlacementBoard`)

## 1. 목적

원장이 강의실별 배치표에서 **이탈 없이** 미납 관리를 끝낼 수 있게 한다.
학생 이름을 클릭하면 빠른 패널이 열려 보호자 연락처·미납 내역을 확인하고,
미납 문자를 보내거나 원생 상세로 바로 이동할 수 있다.

## 2. 범위

- 배치표 카드의 **모든 학생** 이름 클릭 시 빠른 패널 오픈
  - 미납/부분납: 미납 내역·총액·연체일수·미납 문자 발송 강조
  - 완납/환불/미청구: 연락처·결제상태만, 미납 섹션은 "미납 내역 없음"
- 보호자 연락처(모/부/기타) + 복사·전화(tel:) 버튼
- 미납 문자 발송: 데모(`logConsultation` 상담이력 기록), 미납 전용 템플릿·편집 가능
- "상세보기" → `/students?detail=<원생ID>` 딥링크로 원생 상세 모달 자동 오픈

비범위(YAGNI): 실제 SMS 발송, 결제(완납) 처리, 신규 인보이스 생성.

## 3. 컴포넌트 구조

| 파일 | 역할 |
|---|---|
| `lib/payment-helpers.ts` (신규) | 순수 함수 — 미납 내역 조회·총 미납액·연체일수·보호자 연락처 집계 |
| `components/schedule/StudentQuickPanel.tsx` (신규, 재사용 가능) | 원생 빠른 패널 모달 (정보 보기 ↔ 문자 작성 인라인 전환) |
| `components/schedule/PlacementBoard.tsx` (수정) | 칩 이름 텍스트 onClick → 패널 오픈, 드래그와 분리 |
| `app/(admin)/students/page.tsx` (수정) | `?detail=<원생ID>` 딥링크 핸들러 추가 (classes 페이지 선례 복제) |

### 경계/인터페이스
- `StudentQuickPanel`은 mock-data(invoices/guardians/classes/students)를 직접 import(기존 `SmsComposeModal` 패턴과 동일). props는 `{ student, onClose, onToast }`로 최소화.
- 상세보기 이동은 `next/navigation`의 `useRouter().push`로 처리.
- 미납 계산 로직은 전부 `lib/payment-helpers.ts`의 순수 함수로 분리해 단위 테스트 가능하게 한다.

## 4. payment-helpers.ts (순수 함수)

```ts
// 미납/부분납 인보이스 목록 (해당 학생)
unpaidInvoicesOf(studentId: string, invoices: Invoice[]): Invoice[]

// 인보이스 미수금 = tuition + material + content - discount
outstandingAmount(inv: Invoice): number

// 연체 일수 = today - due_date (음수면 0)
overdueDays(dueDate: string, today: string): number

// 학생 보호자 연락처 집계 (모/부/기타) — guardians + student 필드
guardianContactsOf(studentId, students, guardians): { relation: string; phone: string }[]
```

부분납의 미수금 계산은 데모 단계에서 인보이스 총액 기준으로 표시한다(부분 수납액 차감 데이터는 현재 모델에 없음 — 향후 Payment 합산으로 확장 가능, 본 스펙 비범위).

## 5. 패널 UI (위 → 아래)

1. **헤더**: 이름 · 학년 · 반/담당 · 결제상태 배지
2. **보호자 연락처**: 모/부/기타 줄마다 `복사` · `전화` 버튼
   - 주의: 데모 데이터는 전화번호가 모두 더미(010-1234-5678)로 마스킹됨. UX는 동작하나 표시 번호는 동일.
3. **미납 내역** (미납/부분납일 때만): 청구월 · 금액 · 납기일 · `연체 N일` 행 + **총 미납액** 강조
4. **액션**: `미납 문자 발송`(미납 시만) · `상세보기`(항상)

### 미납 문자 발송 (인라인 전환)
- `미납 문자 발송` 클릭 → 같은 모달 안에서 작성 뷰로 전환(중첩 모달 회피)
- 템플릿(편집 가능):
  `[디랩] {이름} 학생 학부모님, {청구월} 수강료 {총액}원이 미납되었습니다. 납기일 {납기일} 확인 부탁드립니다.`
- `발송` → `logConsultation(studentId, message, TODAY)` + 토스트 → 정보 뷰 복귀

## 6. 데이터 흐름

```
칩 이름 클릭 (stopPropagation)
  → PlacementBoard: setQuickStudent(student)
  → StudentQuickPanel 렌더
      ├ payment-helpers로 미납 내역·연락처 계산
      ├ 미납 문자: logConsultation (데모 기록) → onToast
      └ 상세보기: router.push(`/students?detail=${student.id}`)
  → students/page 마운트: ?detail 읽어 setDetailStudent + 모달 오픈
```

## 7. 드래그 충돌 방지

- 칩 본체는 기존대로 `draggable` 유지.
- 이름 `<span>`에 `onClick`(+`stopPropagation`) 부여. 네이티브 드래그가 발생하면 click은 발생하지 않으므로 이동과 충돌하지 않음.

## 8. 엣지 케이스

- 인보이스 없음(미청구) / 완납 / 환불: 미납 섹션 "미납 내역 없음", `미납 문자 발송` 숨김.
- 보호자 연락처 없음: 해당 줄 생략, 전부 없으면 "등록된 연락처 없음".
- 같은 학생 다중 미납월: 내역을 리스트로 표시(현재 데이터는 1건이나 리스트 구조로 향후 대비).

## 9. 테스트

- vitest 단위 테스트 (`lib/payment-helpers.test.ts`):
  - `unpaidInvoicesOf`: 미납/부분납만 필터, 완납 제외
  - `overdueDays`: 납기 경과/미경과/당일 경계
  - `outstandingAmount`: 할인 차감
  - `guardianContactsOf`: 모/부/기타 집계, 없음 처리
- 컴포넌트는 경량 스모크(렌더/버튼 노출 조건) 수준.

## 10. 재사용

`StudentQuickPanel`과 `payment-helpers`는 결제/대시보드 화면에서도 동일하게 재사용 가능하도록 schedule 의존성을 두지 않는다.
