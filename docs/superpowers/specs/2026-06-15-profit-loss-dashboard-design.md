# 손익 대시보드 설계

작성일: 2026-06-15
대상 화면: `app/(admin)/dashboard/page.tsx`
관련 데이터: `lib/mock-data.ts`

## 배경 & 목적

현재 `매출 대시보드`는 학원 내부에서 발생하는 매출(교육비·교구 대여비·콘텐츠 사용비)과
결제수단·결제율·미납·출결만 보여주며, 모든 값을 단일 성격("매출")으로 다룬다.

가맹점주가 월별로 손익을 한눈에 파악하려면 학원 매출 외에 다음 항목이 포함되어야 한다.

- 본사에 지급하는 **분기별 로열티** (비용)
- **교안 마켓플레이스 수익** (매출)
- **교안 마켓플레이스 이용료** (비용)

성격이 다른 수입/비용을 한 화면에서 명확히 구분해 보여주는 것이 이 작업의 핵심이다.

## 결정 사항 (브레인스토밍 합의)

1. **구조: 손익 요약형.** 상단 KPI를 `총수입 · 총비용 · 순이익` 3장으로 교체한다.
   ("이번 달 실제로 얼마 남았나"를 최상단에서 즉시 확인)
2. **화면 제목:** `매출 대시보드` → `손익 대시보드`.
3. **로열티 반영 방식: 현금주의.** 분기 로열티는 **청구되는 달에만 전액** 비용으로 계상하고
   나머지 달은 0. 데모 기준 달인 2026년 6월을 분기 정산월로 가정한다.
   로열티가 잡히는 달은 순이익 급락이 오해되지 않도록 안내 문구/배지를 함께 표기한다.
4. **교안 마켓플레이스: 합산형 + 별도 카드.** 수익/이용료를 별도 카드(보라색)로 묶어 보여주되,
   그 값은 상단 총수입/총비용/순이익에 **합산**된다(드릴다운 성격).
   단, 결제수단(카드/현금/계좌이체) 집계에서는 **제외**한다.
5. **기존 섹션 유지:** 결제 수단별 매출, 원생별 결제 현황·미납자 목록·문자 발송, 오늘 출결 요약은
   현재 위치와 동작을 그대로 둔다.

## 화면 구성 (위→아래)

### 1. 헤더
- 제목 `손익 대시보드`, 부제 `판교 캠퍼스 · 2026년 6월`.
- 우측 월 선택 Select(기존 유지).

### 2. 상단 KPI 3장 (기존 3장 교체)

| 카드 | 값(예시) | 색 | 보조 텍스트 |
|---|---|---|---|
| 총수입 | 13,320,000원 | 초록(`#0F7B6C`) | `+480천 vs 지난달` 등 증감 |
| 총비용 | 1,590,000원 | 빨강(`#EB5757`) | `분기 로열티 포함` (청구월일 때) |
| 순이익 | **11,730,000원** | 기본색, 강조 테두리 | `총수입 − 총비용` |

### 3. 수입 / 비용 항목 분해 (2열 카드)

- **수입 항목** (초록 계열 막대):
  - 교육비 10,320,000
  - 교구 대여비 1,860,000
  - 콘텐츠 사용비 660,000
  - 교안 마켓 수익 480,000 — `신규` 표식
- **비용 항목** (빨강 계열 막대):
  - 본사 로열티 1,500,000 — `분기 청구` 배지 (청구월에만 노출)
  - 교안 마켓 이용료 90,000
  - 평소 달 안내: 로열티 없이 마켓 이용료만 계상됨

각 항목은 기존 "항목별 매출"의 막대 + 금액 + 비율 패턴을 재사용한다.

### 4. 교안 마켓플레이스 카드 (별도, 보라색 계열)

- 수익 +480,000 · 이용료 −90,000 · **순익 +390,000**
- "위 총수입/총비용에 이미 포함됨" 안내 문구.
- 결제수단 집계에는 미포함.

### 5. 기존 섹션 (변경 없음)
- 결제 수단별 매출 (카드/현금/계좌이체) — 학원 매출만 집계.
- 원생별 결제 현황 · 결제율 막대 · 미납자 목록 · 결제 문자 발송 모달.
- 오늘 출결 요약 3장.

## 데이터 모델 (`lib/mock-data.ts`)

`dashboardData`에 손익 관련 필드를 추가/정리한다. 예시 값:

```ts
// 수입
tuition_revenue:     10_320_000,
material_revenue:     1_860_000,
content_revenue:        660_000,
marketplace_revenue:    480_000,   // 교안 마켓 수익 (신규)
total_revenue:       13_320_000,   // 위 합계

// 비용
hq_royalty:           1_500_000,   // 본사 분기 로열티
royalty_is_billing_month: true,    // 이번 달이 분기 정산월인가
marketplace_fee:         90_000,   // 교안 마켓 이용료
total_cost:           1_590_000,   // 위 합계 (청구월 아니면 로열티 제외)

// 손익
net_profit:          11_730_000,   // total_revenue − total_cost

// 교안 마켓 별도 손익 (드릴다운)
marketplace_net:        390_000,   // marketplace_revenue − marketplace_fee
```

- `marketplace_revenue`/`marketplace_fee`는 결제수단 집계(`card_revenue` 등)와 무관하게 별도 관리.
- `prev_month_revenue`/`revenue_diff`는 총수입 기준 증감으로 의미를 유지한다.
- 비청구월 동작: `royalty_is_billing_month`가 false면 `total_cost`에서 로열티를 제외하고
  `분기 청구` 배지/안내를 숨긴다.

## 비범위 (YAGNI)

- 실제 PG/회계 연동, 월별 시계열 차트, 다년 비교는 이번 범위에 포함하지 않는다.
- 월 선택에 따른 다른 달 실데이터 구성은 하지 않는다(6월 단일 데모 유지).
- 로열티 발생주의(1/3 분산) 옵션은 채택하지 않음(현금주의 확정).
