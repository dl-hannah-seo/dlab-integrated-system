# 교안 마켓플레이스 — 설계 스펙

- **작성일**: 2026-06-16
- **상태**: 승인됨 (구현 대기)
- **범위**: 사이드바 진입점 + LMS 스토어프론트 랜딩 페이지 + 기본 장바구니 (목업)

## 1. 배경 / 목적

D.LAB OS 사이드바 하단(키오스크 링크 옆)에 **교안 마켓플레이스** 진입점을 새 탭으로 추가한다.
마켓플레이스는 단순 커머스가 아니라 **실제 수강이 이뤄지는 LMS**다 — 교안·강의를 구매하면
학생이 그 안에서 수업을 듣거나 교재로 활용한다. 따라서 화면은 "쇼핑몰"이 아니라
**"검증된 학습 콘텐츠를 비교·선택하는 학습 플랫폼"** 으로 보여야 한다.

이번 작업의 범위는 **랜딩(스토어프론트) + 기본 장바구니까지**다. 실제 결제 연동,
로그인, 수강(플레이어), 상세 페이지는 후속 과제로 분리한다.

## 2. 디자인 방향 — "방향 A: 인프런형 (신뢰·정보 밀도)"

3개 레퍼런스(인프런·Udemy·클래스101) 카드 패턴 조사 결과를 토대로 **방향 A**를 채택한다.

- **채택 근거**: 대상이 코딩 학원의 학생/학부모다. 학부모는 "검증된 커리큘럼"을 빠르게
  비교하려 하므로, 평점·수강생 수·스킬 태그·난이도 등 **신뢰 신호를 카드에서 바로 노출**하는
  정보 밀도형이 적합하다. 기존 D.LAB OS의 Notion풍 고밀도 정보 UI와도 톤이 맞는다.
- **배제**: 클래스101형(큰 썸네일·감성·가격 숨김)은 둘러보는 재미는 있으나 정보 비교가
  어렵다(레퍼런스에서 명확히 비판받은 안티패턴). 구매가 목적인 이번 화면에는 부적합.
- **캐러셀 제외**: 하이브리드(상단 큐레이션 캐러셀)도 검토했으나, 이번에는 순수 A —
  깔끔한 단일 카드 그리드로 간다. 캐러셀은 후속 확장 여지로만 남긴다.

## 3. 라우트 / 파일 구조

표준 Next.js App Router. 키오스크와 동일하게 **admin 사이드바가 없는 독립 화면**으로 둔다.

- `app/marketplace/layout.tsx` — `metadata`(타이틀) + 페이지 스코프 래퍼
- `app/marketplace/page.tsx` — `'use client'` 단일 페이지 (그리드 + 장바구니 상태)
- 사이드바 진입점은 이미 추가됨: `components/layout/Sidebar.tsx`의 `/marketplace`
  링크(`target="_blank"`). 본 작업으로 해당 링크가 실제 화면에 연결된다.

## 4. 화면 레이아웃

```
┌─────────────────────────────────────────────────────────┐
│  교안 마켓플레이스 (워드마크)              🛒 장바구니 (2) │  ← 상단 바
├─────────────────────────────────────────────────────────┤
│  타이틀 / 부제                                            │
│  [전체] [파이썬] [아두이노·로보틱스] [AI·데이터] [웹·앱]   │  ← 카테고리 탭
├─────────────────────────────────────────────────────────┤
│  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐            │
│  │ 카드   │ │ 카드   │ │ 카드   │ │ 카드   │   (3~4열)   │  ← 카드 그리드
│  └────────┘ └────────┘ └────────┘ └────────┘            │
└─────────────────────────────────────────────────────────┘
```

- 상단 바: "교안 마켓플레이스" 워드마크 + 🛒 장바구니 버튼(담은 개수 배지)
- 카테고리 탭: `전체 / 파이썬 / 아두이노·로보틱스 / AI·데이터 / 웹·앱`
  - 선택 카테고리에 따라 그리드 필터링 (로컬 상태)
- 카드 그리드: 반응형 3~4열

### 카드 해부도 (위 → 아래)

1. **썸네일** (16:9, 라운드, 이모지 placeholder) + 좌상단 *유형 배지*(`교안`/`강의`),
   우상단 *난이도*(입문/초급/중급). `badge`(베스트/신규)가 있으면 함께 표기.
2. **제목** (2줄 클램프)
3. **메타 줄**: 강사 · 차시 수 · 총 강의시간
4. **신뢰 신호**: ⭐ 평점 + (수강생 N명)
5. **스킬 칩**: `Python`, `아두이노` 등
6. **가격 + "담기" 버튼**

## 5. 장바구니 (기본 기능)

- **우측 슬라이드오버 패널** (오버레이 + 우측에서 슬라이드 인)
- 기능: 담기 / 수량 ± / 항목 삭제 / **합계 자동 계산**
- "결제하기" → **목 결제 확인 모달** (실제 결제 연동 없음)
- 동일 항목 재담기 시 수량 증가
- 상태: 페이지 로컬 `useState` (목업이라 새로고침 시 초기화)

## 6. 데이터 모델

`lib/mock-data.ts`에 기존 `Product` 패턴과 동일한 스타일로 추가한다.

```ts
export type LessonType = '교안' | '강의';
export type LessonLevel = '입문' | '초급' | '중급';

export interface LessonProduct {
  id: string;
  title: string;
  category: string;          // '파이썬' | '아두이노·로보틱스' | 'AI·데이터' | '웹·앱'
  type: LessonType;
  instructor: string;
  level: LessonLevel;
  lessons: number;           // 차시 수
  durationMin: number;       // 총 강의시간(분)
  rating: number;            // 0~5 (소수 1자리)
  students: number;          // 수강생 수
  skills: string[];          // 스킬 칩
  priceWon: number;          // 원
  thumbnailEmoji: string;    // 썸네일 placeholder
  badge?: '베스트' | '신규';
}

export const lessonProducts: LessonProduct[] = [ /* 6~8개 샘플 */ ];
```

샘플 6~8개는 카테고리/유형/난이도/배지를 골고루 섞어 그리드·필터·배지 렌더가
모두 시연되도록 구성한다.

## 8. 컴포넌트 경계

- `app/marketplace/page.tsx` — 페이지 셸: 카테고리 필터 상태 + 장바구니 상태 보유,
  카드 그리드와 장바구니 패널 조립
- 카드 / 장바구니 패널 / 결제 확인 모달은 같은 파일 내 하위 컴포넌트 또는 인라인으로 구성
  (키오스크 `shop/page.tsx` 패턴 참조 — 단일 파일에 모달 인라인)
- 데이터는 `mock-data.ts`에서 import (페이지에 하드코딩하지 않음)

## 9. 범위 밖 (YAGNI — 후속 과제)

- 검색 / 정렬
- 로그인 / 사용자별 장바구니 영속화
- 실제 결제(PG) 연동
- 강의 상세 페이지
- 수강(플레이어) / "내 강의실"
- 상단 큐레이션 캐러셀

## 10. 시연 시나리오 (수용 기준)

1. 사이드바에서 "교안 마켓플레이스" 클릭 → 새 탭에서 `/marketplace` 열림
2. 카테고리 탭 전환 시 그리드가 해당 카테고리로 필터됨
3. 카드의 "담기" 클릭 → 장바구니 배지 카운트 증가
4. 장바구니 열기 → 수량 ±, 삭제, 합계가 정확히 반영됨
5. "결제하기" → 목 결제 확인 모달 노출
