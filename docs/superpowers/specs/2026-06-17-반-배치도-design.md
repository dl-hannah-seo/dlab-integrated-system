# 반 배치도 (Class Placement Board) — 설계 문서

작성일: 2026-06-17

## 배경 / 목적

랩장님이 "강의실 위치 · 반/과목별 학생 명단이 **한 화면에** 보이고, 학생을 **드래그앤드롭으로 반 이동**시키며, **결제완료** 여부까지 한눈에 보이는" 운영 보드를 원함. 참고 자료는 스프레드시트의 `시간표(상세)` — 토요일 하루를 `강의실 × 시간대`로 펼쳐 각 반의 학생 명단까지 표시한 형태.

### 기존 화면과의 구분
- **기존 [시간표](../../../app/(admin)/schedule/page.tsx)**: *시간 축*(요일×시간대) 기준 주간 캘린더. 반 블록만 표시, 학생 명단·강의실 위치는 안 보임.
- **반 배치도(신규)**: *강의실 축*(강의실 × 시간대) 기준. 각 반의 학생 명단까지 펼쳐 보여주는 "배치판" + 학생 이동/결제상태 운영 도구.

두 화면은 같은 데이터(반·수강생)를 **다른 축**으로 본다. 따라서 별도 메뉴가 아니라 시간표의 두 번째 보기로 둔다.

## 이름 / 위치 (확정)
- **이름**: `반 배치도`
- **위치**: 시간표 메뉴 하위 보기. 사이드바 항목은 추가하지 않음.
- 시간표 페이지 상단에 뷰 토글: `주간 보기` ↔ `강의실별 배치`. 기본값은 기존 `주간 보기`.
- 라우트는 `/schedule` 유지. 토글 상태로 전환(URL 동기화 `?view=board`는 선택).

## 화면 구조

```
[ 주간 보기 | 강의실별 배치 ]            ← 뷰 토글 (페이지 상단)
[ 토 | 화·목 | 월·수·금 ]   [결제완료만][미납만]   ← 요일 선택 + 결제 강조 토글

          1강의실     2강의실     3강의실     …     미배정
 09:00   [반 카드]   [반 카드]   [반 카드]
 10:00   [반 카드]              [반 카드]
  …
```

- **요일 선택**: 캠퍼스 운영 day_group(`토` / `화목` / `월수금`)에서 하나 선택 → 그 요일 하루를 채움. 기본값은 오늘이 포함된/가장 가까운 운영 요일.
- **행 = 시간대**: 09:00~18:00 고정 축(기존 시간표의 `TIME_AXIS` 재사용). 축에 없는 시간대 반이 있으면 정렬해 추가.
- **열 = 강의실**: 해당 요일에 등장하는 강의실들. 강의실 미배정 반은 맨 끝 **"미배정"** 열에 모음(배치 누락 방지).
- **셀 = 반 카드**: 같은 강의실·시간대에 편성된 반.
  - 헤더: 과정명 · 담임 · `인원/정원`
  - 본문: 그 반 수강생 **칩 목록** — 칩에 이름 + 결제 상태 점.

## 데이터 모델

### 추가
- `Class`에 `room?: string` 추가 (고정 강의실, **자유 입력**).
  - **반 관리 생성/편집 모달**에 "강의실" 텍스트 입력 칸 추가. 배치도는 이 값으로 열을 채움.
  - 값이 없으면 "미배정" 열로.

### 재사용(추가 없음)
- 학생 명단: `enrollments`(ended_at === null) → `students`. (반 관리와 동일.)
- 결제 상태: 현재 청구월 `Invoice`를 `(student_id, class_id)`로 조회해 `status` 사용.
  - 🟢 `완납` · 🔴 `미납` · 🟡 `부분납` · ⚪ `환불`/인보이스 없음.
  - "현재 청구월"은 선택 요일 기준 해당 월(`billing_month`)로 해석. 인보이스가 없으면 회색(미청구).

## 상호작용

### 학생 드래그 이동 (반 A → 반 B)
- 칩을 다른 반 카드로 드롭하면 **가벼운 확인 모달** 표시:
  > `홍길동` 학생을 `토 09:00 · 파이썬기초`에서 `토 10:00 · 알고리즘`(으)로 이동하시겠습니까?
- 확인 시: 반 A 퇴반(`ended_at` 기록) + 반 B 입반. [반 관리](../../../app/(admin)/classes/page.tsx)의 `handleEnroll`/`handleUnenroll`와 **동일한 로직을 공유 유틸로 추출해 재사용**.
- **정원 가드**: 대상 반이 정원이면 드롭 거부(모달 대신 토스트/시각 표시), 이동 안 함.
- 같은 반 안 드롭은 무시.

### 결제 강조 토글
- `결제완료만` / `미납만`: 해당 상태 학생만 강조(나머지는 흐리게). 필터 제거가 아니라 강조 — 전체 배치 맥락은 유지.

## 상태 관리 (프로토타입)
- mock 데이터 기반이므로 배치도는 자체 로컬 state로 `enrollments`(및 필요시 `classes.enrolled_count`)를 변경. 기존 반 관리 페이지와 동일한 프로토타입 패턴.
- 실제 백엔드 연동은 이후 과제. (이 문서 범위 밖.)

## 구조 / 파일 경계

| 파일 | 역할 |
|------|------|
| `lib/placement-board.ts` (신규) | 순수 로직: 요일 → `강의실 × 시간대` 그리드 생성, 반별 학생 명단 + 결제 상태 해석, 드롭 시 입/퇴반 계산. `lib/sessions.ts`·`lib/kiosk-points.ts`와 같은 패턴. |
| `components/schedule/PlacementBoard.tsx` (신규) | 배치도 UI(그리드·반 카드·학생 칩·드래그·확인 모달). |
| `app/(admin)/schedule/page.tsx` (수정) | 상단 뷰 토글 추가. 주간 보기 ↔ `PlacementBoard` 스위치만. 페이지 비대화 방지. |
| `app/(admin)/classes/page.tsx` (수정) | 생성/편집 모달에 "강의실" 입력 추가. 입/퇴반 로직을 공유 유틸로 추출. |
| `lib/mock-data.ts` (수정) | `Class.room?: string` 추가. (선택) 시연용 일부 반에 room 값 부여. |

### 경계 점검
- `lib/placement-board.ts`: 입력은 (요일, classes, enrollments, students, invoices), 출력은 렌더용 그리드 구조. UI 없이 단위 테스트 가능.
- `PlacementBoard`: 그리드 데이터를 받아 그리기 + 사용자 입력을 콜백으로 위임. 내부를 바꿔도 페이지는 안 깨짐.
- 입/퇴반 로직 공유 추출로 반 관리와 배치도의 동작 일관성 보장.

## 범위 밖 (YAGNI)
- 반 블록 자체의 강의실/시간대 드래그 재배치(반 이동) — 학생 이동만 드래그. 강의실은 반 설정에서 텍스트로 변경.
- 실제 백엔드/DB 연동, 결제 데이터 실시간 동기화.
- 주간(여러 날) 동시 배치 — 한 번에 한 요일.
- 강의실 마스터 관리 화면(고정 목록) — 자유 입력으로 충분.
