# 출결 현황 페이지 대시보드 재구성 — 설계

날짜: 2026-06-17
대상 페이지: `app/(admin)/attendance/page.tsx`

## 목적

출결 현황 페이지를 "당일 현황 요약 → 시계열 추이 → 미등원 집중 리스트"로 이어지는
대시보드 상단부와, 그 아래 기존 반별 상세 조회로 재구성한다. 운영자가 페이지를
열었을 때 (1) 오늘 상태를 한눈에, (2) 최근 추세를, (3) 지금 챙겨야 할 학생을
바로 파악하고, 필요 시 아래에서 반별로 깊게 조회하는 흐름을 만든다.

## 전체 레이아웃 (위 → 아래)

```
[1행] 당일 현황 원그래프 (좌)   |   미등원 집중 리스트 (우)
[2행] 주/월간 출석률 추이 (전폭)
[3행] 반별 상세 조회 ── 필터 드롭다운 + 반별 카드 + 기록부 모달 (기존 유지)
```

- 1행: 2열 배치. 좌측 원그래프, 우측 집중 리스트.
- 2행: 기존 `AttendanceTrend` 전폭.
- 3행: 기존 반별 상세 조회(필터 + 카드 리스트 + `ClassRecordModal`) 위치만 하단으로.

## 위젯 설계

### ① 당일 현황 원그래프 (기존 KPI 카드 4개를 완전히 대체)

- 기존 `components/ui/DonutChart.tsx`를 재활용한다.
- 슬라이스 3종:
  - 출석(보강 포함) — `#0F7B6C`
  - 미도착(pending) — `#C7C6C3`
  - 결석 — `#EB5757`
- `DonutChart`에 **선택적 중앙 라벨(center label)** prop을 추가하여 도넛 가운데에
  당일 출석률(%)을 큼직하게 표시한다. "당일 현황 요약 원그래프"의 핵심 정보.
- 범례는 컴포넌트 기본 동작 그대로 항목별 % 표시. `showAmounts=false`이므로 "원"
  표기는 나오지 않는다.
- 데이터는 기존 페이지의 `todayRecords`/`kpi` 계산(오늘 회차 = `session_id`가
  `sess-`로 시작)을 그대로 사용한다. 출석률 분모는 미도착 제외(출석+결석).
- KPI 카드 4개(출석/미도착/결석/출석률)는 제거한다. 수치는 원그래프 범례와 중앙
  라벨이 모두 담는다.

### ② 미등원 집중 리스트 (신규)

- 신규 컴포넌트 `components/attendance/AbsenceFocusList.tsx`.
- 신규 헬퍼 `getAbsenceFocusList(records, classIds, maxSessions = 8)`를
  `lib/mock-data.ts`에 추가한다.
  - 현재 학기 반들(`classIds`)의 **최근 8회차** 기준으로 학생별 결석 횟수를 집계.
  - **오늘 진행 중인 회차(`session_date === TODAY`)는 제외**한다(미완료라 왜곡 방지,
    추이 집계와 동일한 규칙).
  - **결석 횟수 내림차순** 정렬. **결석 1회 이상**인 학생만 노출.
  - 반환 항목: 학생, 소속 반, 결석 횟수, 집계 대상 회차 수, (보조) 최근 결석일.
- 행 구성: 학생명 · 소속 반(schedule/course) · `결석 N회` 배지 · (보조) 최근 결석일.
- 행 클릭 → 그 학생이 속한 반의 `ClassRecordModal`을 연다(기존 `openClass` state
  재사용). 결석 맥락을 매트릭스에서 바로 확인.
- 결석 1회 이상 학생이 없으면 빈 상태 문구를 표시한다.

### ③ 주/월간 추이 (변경 없음)

- 기존 `components/attendance/AttendanceTrend.tsx`를 그대로 사용, 2행 전폭 배치.

### ④ 반별 상세 조회 (변경 없음)

- 기존 반 필터 드롭다운 + 반별 카드 리스트 + `ClassRecordModal` 로직을 그대로 두고
  위치만 대시보드 아래로 옮긴다.

## 데이터/계산 규칙 요약

- 당일 집계: `session_id`가 `sess-`로 시작하는 레코드. 출석률 = 출석/(출석+결석),
  미도착 제외.
- 집중 리스트: 최근 8회차(오늘 제외)에서 결석 횟수 집계, 내림차순, 결석 ≥ 1만 노출.
- 출석=`attend` 또는 `makeup`로 취급(기존 페이지 규칙과 동일).

## 비범위 (YAGNI)

- 추이 차트 로직·디자인 변경 없음.
- 반별 상세 조회/기록부 모달 동작 변경 없음.
- 결석률 퍼센트 정렬, 임계값 필터 등 대안 선별 방식은 채택하지 않음(결석 횟수 순만).
- 학생 상세 페이지로의 별도 네비게이션은 만들지 않음(반 기록부 모달로 통일).
