# 출결현황(/attendance) 개선 설계 스펙

> 작성일: 2026-06-16 · 대상: `app/(admin)/attendance/page.tsx` · 단계: Phase 1 시연용

## 목표

출결현황 페이지를 "읽기 전용 요약"에서 **분석 대시보드(A) + 반별 기록부 드릴다운(B)** 으로 발전시킨다. 세 가지 요청을 모두 충족한다:

1. **추이** — 주간/월별 출석률 추이 (실제 mock 이력 집계)
2. **반별 상세 조회** — 반 클릭 시 학생×회차 매트릭스
3. **편집** — 매트릭스 칸을 직접 클릭해 출결 상태 정정

## 결정 사항 (브레인스토밍 결과)

| 항목 | 결정 |
|------|------|
| 화면 성격 | A(분석 대시보드) 중심, B(기록부) 병행 |
| A↔B 연결 | **드릴다운** — 반별 카드 클릭 → 반별 기록부 모달 |
| 추이 차트 | **안 A: 출석률 막대** + `주간/월별` 토글 |
| 편집 방식 | **안 A: 매트릭스 직접 편집** — 칸 클릭 → 상태 팝오버 |
| 편집 저장 | **클라이언트 상태만** (새로고침 시 초기화, 데모용) |
| 이력 범위 | **전 반·회차 전체** 결정적 백필 |

## 비목표 (YAGNI)

- 백엔드 영속화·DB 연동 (데모는 클라이언트 상태)
- 전체 반 통합 기록부 탭 (드릴다운으로 충분, 추후 확장 가능)
- 막대+선 콤보/누적 구성 차트 (단순 출석률 막대로 확정)
- 학생별 출결 이력 별도 페이지 (원생 상세에서 별도 처리)

## 1. 메인 페이지 — 분석 대시보드

기존 페이지 구조를 유지·발전시킨다. 위→아래 순서:

### 1-1. 오늘 요약 KPI (유지)
출석 / 미도착 / 결석 / 출석률 4칸. 현재 구현 유지. 단, 편집으로 상태가 바뀌면 즉시 재집계되도록 집계 소스를 공유 상태로 변경.

### 1-2. 출석률 추이 (개선)
- 카드 제목 우측에 `주간 / 월별` 토글 (세그먼트 버튼).
- **주간**: 최근 8주, 각 주의 캠퍼스 전체 출석률 막대.
- **월별**: 최근 4개월, 각 월 출석률 막대.
- 현재 막대 스타일(주황 `#FF6C37`, 최신 강조) 유지. 하드코딩된 `weeklyStats` 제거 → 이력 집계 함수로 대체.
- 막대 위 퍼센트, 막대 아래 기간 라벨. 최신 기간만 불투명도 1, 과거는 0.4.

### 1-3. 반별 카드 리스트 (개선)
- 반별 요약 줄: 시간표·과정·담당 + 출석/미도착/결석/출석률. 현재 유지.
- 변경점: **카드 전체가 클릭 가능** → 반별 기록부 모달 열림. hover 시 커서·미세 배경 변화로 클릭 가능 암시.
- 명단 점(dot) 미리보기는 카드에 그대로 두되, "자세히 보기" 진입점 역할.

## 2. 반별 기록부 모달 — 드릴다운 + 편집

기존 `components/ui/Modal.tsx`의 `size="xl"`(max-w-4xl) 재사용.

### 2-1. 헤더·요약
- 제목: `{schedule} {course} · 담당 {teacher}` (예: "토 09:00 파이썬 기초 · 담당 론").
- 상단 한 줄 요약: 그 반의 회차별 평균 출석률 / 총 회차 수 / 재원 인원.

### 2-2. 학생 × 회차 매트릭스
- **행** = 반 소속 학생 (이름 고정 좌측 열).
- **열** = 회차 (1회 ~ 현재 회차). 열 헤더에 회차 번호·날짜.
- **셀** = `AttendanceDot`(기존 컴포넌트) 상태색: 출석(녹) / 보강(노) / 결석(적) / 미도착(회).
- 오늘 회차 열은 테두리·배경으로 강조.
- 회차가 많으면 가로 스크롤 (모달 본문 `overflow-x-auto`).
- 범례: ●출석 ●보강 ●결석 ●미도착 + ▢오늘 + 수정됨 표식.

### 2-3. 셀 편집 (팝오버)
- 셀 클릭 → 작은 팝오버: **출석 / 결석 / 보강 / 미도착** 선택지.
- **결석** 선택 시 사유 입력 필드(선택 입력) 노출.
- 선택 즉시: 해당 출결 레코드 `status`(필요시 `absence_reason`) 갱신 → 매트릭스·반별 요약·메인 KPI·추이 모두 재계산.
- 관리자 수정 레코드는 `source: 'manual'`로 표시하고, 셀 모서리에 미세한 "수정됨" 점(dot) 표식.
- 팝오버는 바깥 클릭·ESC로 닫힘. 한 번에 한 셀만 편집.

## 3. 데이터 모델 & 백필

`lib/mock-data.ts`에 **결정적(deterministic) 출결 이력**을 추가한다. (프로젝트 규칙상 `Math.random()` 금지 → 학생 인덱스·회차 기반 고정 패턴 사용)

### 3-1. 회차 이력 (`sessionHistory`)
- 6개 현재 반(cl-01~cl-06) 각각에 대해 회차 생성.
- 토반(cl-01·02·03): 주 1회(토). 화목반(cl-04·05·06): 주 2회(화·목).
- 주간 8주 / 월별 4개월 추이를 채울 만큼 과거로 백필. 가장 최신 회차 = 오늘(2026-06-14).
- 기존 `todaySessions`·`initialAttendance`(cl-01 오늘치)는 **그대로 두고**, 오늘 열은 그 상태를 재사용해 키오스크 데모와 충돌 방지.

### 3-2. 출결 이력 (`attendanceHistory`)
- 각 (회차 × 소속 학생)마다 출결 레코드 생성.
- 상태 분포: 대부분 출석, 소수 결석·보강 — 학생 streak·인덱스에 비례한 고정 규칙으로 산출(높은 streak일수록 출석률↑). 캠퍼스 전체 출석률이 88~96% 대로 떨어지게 튜닝.
- 과거 레코드는 `source: 'kiosk'`, `checked_in_at` 채움. 결석은 일부에 `absence_reason`.

### 3-3. 집계 헬퍼
- `getWeeklyAttendanceTrend(weeks=8)` — ISO 주차별 출석률.
- `getMonthlyAttendanceTrend(months=4)` — 월별 출석률.
- `getClassMatrix(classId)` — 그 반의 학생×회차 출결 2차원 구조.
- 모든 헬퍼는 현재(편집 반영된) 출결 상태를 기준으로 집계.

## 4. 상태 관리 & 격리

- 편집은 백엔드 없이 **페이지 로컬 상태**(React state)로만 반영. 초기값은 `attendanceHistory` + `initialAttendance`.
- 추이·KPI·매트릭스가 모두 같은 출결 상태를 바라보도록, 출결 상태를 페이지 상단에서 한 번 보유하고 하위로 내려준다.
- 매트릭스·팝오버 등 신규 UI는 별도 컴포넌트로 분리해 `page.tsx` 비대화 방지:
  - `components/attendance/AttendanceTrend.tsx` — 추이 카드(토글 포함)
  - `components/attendance/ClassRecordModal.tsx` — 반별 기록부 모달(매트릭스 + 편집)
  - `components/attendance/StatusPopover.tsx` — 셀 상태 변경 팝오버
- 데모 시연 재생(`useAttendanceDemo`, DEMO ONLY)과는 무관 — 그 기능은 빠른 실행 패널 한정이며 본 작업과 분리 유지.

## 5. 검증

프로젝트에 테스트 러너가 없으므로(기존 관례):
- `npx tsc --noEmit` 타입 체크.
- `npm run dev` 수동 스모크: ①추이 주간/월별 토글 ②반 카드 클릭 → 모달 ③매트릭스 셀 클릭 → 상태 변경 → 매트릭스·KPI·추이 즉시 반영 ④결석 사유 입력 ⑤수정됨 표식 노출.
