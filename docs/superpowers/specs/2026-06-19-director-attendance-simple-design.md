# 원장 출결 현황 — 단순 현황 뷰 설계

날짜: 2026-06-19
대상 페이지: `app/(admin)/attendance/page.tsx`

## 목적

원장(`role === '원장'`)이 출결 현황 페이지를 열었을 때 매일 굵직하게 "잘
돌아가나"를 **한눈에 확인**하도록 단순화한다. 원장은 출석을 찍거나 건건이
처리하는 사람이 아니라 출석 데이터에서 상태를 읽는 사람이므로, 교사용 운영
도구(보강 대기·미등원 집중·기록부 편집 흐름)는 원장 화면에서 걷어내고
**오늘 요약 + 반별 현황** 두 가지만 남긴다.

핵심 결정: 단순화는 **원장 역할에만** 적용한다. 교사·SO는 현재 운영 뷰를
그대로 유지한다(보강 대기 목록·미등원 집중 리스트·필터·반별 카드·기록부 모달).

## 적용 대상 / 분기

- `role === '원장'` → 신규 단순 뷰 렌더링.
- `role === '교사' | 'SO'` → 기존 뷰 그대로(변경 없음).
- 기본 역할이 `'원장'`이므로 데모 진입 시 단순 뷰가 먼저 보인다.

## 레이아웃 (위 → 아래)

```
[헤더]  출결 현황 · 판교 캠퍼스
[1행]   KPI 카드 4칸: 오늘 출석률 | 출석 | 미도착 | 결석
[필터]  그룹(연도+학기) 드롭다운 + 반 드롭다운
[2행]   반별 출석 현황 표 (클릭 가능 → 기록부 모달)
          반 / 담당 / 출석 / 미도착 / 결석 / 출석률 / ›
```

## 위젯 설계

### ① KPI 카드 4칸 (참고 이미지 스타일)

- 4칸: **오늘 출석률 · 출석 · 미도착 · 결석**.
  - 오늘 출석률: 큰 수치 `91%`, 보조 라벨 "전체 반 평균".
  - 출석: `7명`, 보조 라벨 "정상 출석".
  - 미도착: `1명`. 결석: `0명`.
- 데이터는 기존 페이지의 `kpi` 계산을 그대로 사용한다.
  - 당일 회차 = `session_id`가 `sess-`로 시작하는 레코드.
  - 출석률 = 출석 / (출석 + 결석). **미도착(pending) 제외**.
  - 출석 = `attend` 또는 `makeup`로 취급(기존 규칙과 동일).
- **지각(`지각`) 상태는 신설하지 않는다.** 참고 이미지엔 지각 칸이 있으나 현재
  데이터 모델(attend/makeup/absent/pending)에 없으므로 미도착으로 대체한다.
- 카드 디자인은 프로젝트 라이트 테마(`#37352F` 텍스트, `#787774` 보조,
  `Card` 컴포넌트)를 따른다. 참고 이미지의 다크 테마는 채택하지 않는다.

### ② 필터 (유지)

- 기존 그룹(연도+학기) 드롭다운 + 반 드롭다운을 원장 뷰에도 그대로 둔다.
- 기존 `groupFilter`/`classFilter` state·`groupOptions`/`classOptions`/
  `targetClasses` 계산 로직을 그대로 재사용한다.

### ③ 반별 출석 현황 표 (신규 — 기존 반별 카드 계산을 표 형태로)

- 컬럼: **반 · 담당 · 출석 · 미도착 · 결석 · 출석률 · ›**
  - 반: `cls.schedule` + `cls.course` (예: "월·수 18:00 알고리즘 A").
  - 담당: `cls.teacher`.
  - 출석/미도착/결석: 그 반 당일 회차 집계 수.
  - 출석률: 출석 / (출석 + 결석). 당일 회차가 없으면 과거 누적으로 대체하고
    "누적" 표기(기존 반별 카드 규칙과 동일).
- 행 집계는 기존 반별 카드의 계산 블록(`attendCount`/`absentCount`/
  `pendingCount`/`pct`, `hasToday` 분기 포함)을 그대로 옮긴다.
- 행 클릭 → 기존 `ClassRecordModal`을 연다(`openClass` state 재사용). 원장이
  이상 반을 발견하면 바로 기록부로 드릴인.
- 표 대상 반은 `targetClasses`(필터 적용된 반 목록)를 사용한다.

## 원장 뷰에서 빠지는 것

당일 도넛(`DonutChart`), 미등원 집중 리스트(`AbsenceFocusList`), 보강 대기
목록, 반별 카드(버튼) UI. (계산 헬퍼·컴포넌트 자체는 교사·SO 뷰가 계속
사용하므로 삭제하지 않는다.)

## 파일 변경 범위

| 파일 | 변경 |
| --- | --- |
| `app/(admin)/attendance/page.tsx` | 원장 분기 렌더링 추가: KPI 카드 4칸 + 필터(재사용) + 반별 표. 교사·SO 경로는 기존 그대로 |

- 신규 컴포넌트 불필요. KPI 카드·표는 페이지 내 인라인, `ClassRecordModal`은 재사용.
- `kpi`, `groupOptions`, `classOptions`, `targetClasses`, `getClassRoster`,
  반별 집계 로직 등 기존 계산을 그대로 활용한다.

## 데이터/계산 규칙 요약

- 당일 집계: `session_id`가 `sess-`로 시작. 출석률 = 출석/(출석+결석), 미도착 제외.
- 반별 행: 당일 회차 있으면 당일 집계, 없으면 과거 누적(`sh-<clsId>-` 회차)으로
  "누적 N%" 표기.
- 출석 = `attend` 또는 `makeup`.

## 비범위 (YAGNI)

- 주간/월간 추세 차트 — 원장 단순 뷰에 넣지 않음.
- 위험 학생(결석 누적) 리스트 — 넣지 않음.
- 지각 상태 신설 — 하지 않음.
- 교사·SO 뷰 변경 — 없음.
- 다크 테마 — 채택하지 않음(프로젝트 라이트 테마 유지).
