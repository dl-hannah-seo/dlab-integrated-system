# 출석 포인트 키오스크 — dlabos 와이드 레이아웃 이식 설계

작성일: 2026-06-17

## 목표

현재 모바일(좁은 `max-w-sm` 카드) 형태인 출석 키오스크를, dlabos 프로토타입의
**와이드 2단 레이아웃**으로 재작성한다. 첨부된 3개 스크린샷(첫 화면 / 번호 입력 후 /
체크인 후 대시보드)과 동일하게 보여야 한다.

## 화면 구성 (확정)

키오스크는 단일 클라이언트 컴포넌트(`app/kiosk/page.tsx`)가 상태에 따라
두 단계를 렌더한다. 상단에는 `출석 키오스크` 제목만 있는 헤더 바
(우측 `역할 전환`·`데모 초기화` 버튼은 **없음**).

### 1단계 — 입력 (`input`)  ← 스크린샷 1·2
와이드 2단: 좌측 키패드 카드 + 우측 사이드 패널.

- **좌측 키패드 카드**: 📲 아이콘, "D.LAB 출석 체크", "전화번호 뒤 4자리를 눌러주세요",
  입력 표시(`_ _ _ _` → 숫자), 3×4 키패드(1~9, `C`/`0`/`←`). `C`·`←`는 오렌지 액션키.
- **우측 사이드 패널**(입력 자리수·매칭 상태에 따라 분기):
  - 입력 전 / 4자리 미만: 👋 "번호를 누르면 학생 정보가 여기에 나와요" +
    테스트 번호 칩(실제 시드 데이터의 뒤 4자리에서 파생, 예: `1842` `3074` `9201`).
  - 4자리 일치(1명 이상): "학생 확인 / 본인이 맞으면 출석을 눌러요" + 학생 카드.
    카드 = 아바타(이름 둘째 글자) + 이름 + `반명 · Lv.N · {points}P` + `[출석 확인]` 버튼.
    여러 명이면 카드 여러 개(본인 선택).
  - 4자리인데 0건: 😅 "일치하는 학생이 없어요 / 번호를 다시 확인해 주세요".

`[출석 확인]` 클릭 → 해당 학생 `points += 30`, `balance += 30` → `done` 단계로 전환.

### 2단계 — 체크인 후 (`done`)  ← 스크린샷 3
와이드 2단(`1.04fr / .96fr`, 최대폭 ~1060px), 좁은 화면에선 1단 스택.

- **좌측 컬럼**
  - 환영 카드: 🎉 "{이름}, 환영해요!" + "{입실시각} 입실 완료 · 오늘도 화이팅! 🚀"
  - 포인트 2카드: `사용 가능 포인트 {balance}P`(⭐ 방금 +30P) / `레벨 포인트 {points}P`(🏆 Lv.N · {랭킹}위)
  - 오늘의 수업 카드: 반명, `{교사} 선생님 · {일정}`, `🎯 「{course}」 미션`, 레벨 진행바, "다음 레벨까지 {N}P"
  - 버튼 2개: **`[처음 화면으로]`**(주: 기존 "다른 친구 출석"에서 명칭 변경 — 누르면 `input` 단계로 초기화) +
    **`[내 대시보드]`**(비활성 — 외형만 유지, 동작 없음)
- **우측 컬럼**
  - 포인트 랭킹: 상위 3명(금/은/동) + 본인이 4위 이하면 본인 행 추가, "(나)" 표시
  - 포인트 상점: 상점 아이템 행(아이콘·이름·`{cost}P`·버튼). `balance >= cost`면 `[교환]`,
    아니면 `[부족]`(비활성). 교환 시 `balance -= cost`(누적 `points`·랭킹은 불변).
    패널 헤더에 `사용가능 {balance}P` 태그.

## 데이터 모델 (`lib/mock-data.ts` 보강)

현재 `Student`는 `points`만 있고 프로토타입은 `balance`/`level`/`leaderboard`/`shopItems`를 쓴다.
최소 보강:

- `Student.balance: number` 추가 — **사용 가능 포인트**(상점 차감 대상).
  기존 `points`는 **누적/레벨/랭킹용**으로 유지.
- 레벨·진행률은 파생값(필드 불필요):
  - `level = Math.floor(points / 300) + 1`
  - `진행률 = (points % 300) / 300`, `다음 레벨까지 = 300 - (points % 300)`
- `leaderboard` — `students`를 `points` 내림차순 정렬해 파생(캠퍼스/반 기준은 구현 시 확정).
- `shopItems` — 상수 배열로 정의: 간식 교환권 200P, 게임 시간 30분 300P, 디랩 노트 150P,
  특별 수업 참가권 500P, 디랩 굿즈 키트 800P (아이콘 포함).
- 데모 학생 정렬: 테스트 번호(`1842`/`3074`/`9201` 등)가 동작하도록 키오스크 시연용
  학생 몇 명의 `parent_phone` 뒤 4자리와 `points`/`balance`를 프로토타입 시드에 맞춰 조정.
  (시연 데이터이므로 정확한 수치는 화면과 어색하지 않은 선에서 맞춘다. 힌트 칩은
  실제 시드 뒤 4자리에서 동적 생성.)

## 인증 (4자리, 확정)

- 키패드 4자리 입력 → `parent_phone` 의 **뒤 4자리** 일치 학생을 필터.
- 단일 매칭이면 카드 1개, 복수 매칭이면 카드 여러 개(본인 선택), 0건이면 안내 문구.
- (참고: 기존 HEAD 코드는 8자리 `parent_phone` 매칭이었으나, 본 작업에서 4자리로 변경.)

## 스타일

- 프로토타입의 와이드 클래스(`.kiosk`, `.celebrate-wide`, 키패드·패널·랭킹·상점)를
  현재 시스템의 다크 테마 CSS 변수(`--kiosk-bg/-surface/-card/-orange/-orange2/-muted/-border/-text`,
  `globals.css`에 기정의)와 Tailwind/인라인 스타일로 이식.
- 색·간격·폰트 굵기는 스크린샷 기준으로 맞춘다. 라벨 위계가 같으면 굵기도 동일하게.
- 반응형: 좁은 화면에서 2단 → 1단 스택(프로토타입 `@media`와 동일 분기).
