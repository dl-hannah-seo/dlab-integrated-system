# 키오스크 — 출석 체크 / 포인트 상점 기능 분리 설계

작성일: 2026-06-17
대상: `app/kiosk/page.tsx` (+ 필요 시 동일 파일 내 보조 컴포넌트)

## 문제

현재 키오스크는 4자리 입력 → 확인 → 학생 선택 → 체크인 후 곧바로 **포인트 상점(교환 버튼 활성) 대시보드**가 뜬다. 등하원 시 부모·친구가 대신 번호를 눌러 출석시킬 수 있는데, 그 상태에서 바로 포인트가 교환돼버릴 수 있다. 빠르고 공개적인 **출석**과, 포인트를 실물과 바꾸는 **상점(거래)** 은 성격이 달라 분리가 필요하다.

## 해결 방향 (확정)

암호적 신원확인이 아니라 **흐름 분리**로 해결한다. 본인 확인은 출석과 동일한 4자리(부모 전화 뒤 4자리 = `kioskCodes`)를 쓰되, 교환은 **출석의 자동 결과가 아니라** 사용자가 '포인트 상점'을 일부러 선택해 코드를 입력했을 때만 가능하게 한다. 출석 플로우에서는 포인트를 절대 쓸 수 없다.

## 화면 구성

### 1) 홈(랜딩) — `mode: 'home'`
- 두 개의 큰 진입 버튼: `📲 출석 체크` / `🛍️ 포인트 상점`
- 키오스크 진입 시 기본 화면.

### 2) 출석 체크 — `mode: 'attend'`
- 좌: 키패드(4자리) + [확인] 버튼 / 우: 학생 리스트(상하 중앙). (기존 입력 단계 UI 재사용)
- 4자리 입력 후 [확인] → 일치 학생 리스트 → 학생 선택 시:
  - 누적 `points += 30`(시연), 사용가능 `balance += 30`
  - **"{이름} 학생 출석 완료! ⭐ +30P" 토스트** 표시
  - 약 **2초 후 자동으로 키패드 첫 화면(빈 입력)으로 복귀** → 다음 학생 즉시 출석 가능
- **대시보드·상점을 일절 노출하지 않음** → 출석만으로 포인트 차감 불가.
- 좌상단/홈으로 돌아가는 경로 1개 제공(예: `← 홈`).

### 3) 포인트 상점 — `mode: 'shop'`
- 동일한 키패드(4자리) + [확인] → 학생 리스트 → 학생 선택 시:
  - 선택 학생을 활성화하고 **상점 대시보드**를 보여준다.
- 상점 대시보드 = **기존 체크인 후 대시보드를 그대로 재사용**:
  - 환영 헤더(🎉 플로팅) / 사용 가능·레벨 포인트 2카드 / 오늘의 수업 / 포인트 랭킹 / 포인트 상점(**교환 활성**)
  - 단, 출석이 아니므로 **"⭐ 방금 +30P" 배지는 제거**(사용 가능 포인트 카드).
  - 환영 문구는 맥락에 맞게 유지/조정(예: "{이름}, 환영해요!").
- 교환(`교환` 버튼): `balance >= cost`면 `balance -= cost`(시연). 누적 `points`·랭킹 불변.
- 버튼: `처음 화면으로`(상점 키패드로) / `홈`(랜딩). `내 대시보드`는 기존대로 비활성.

## 인증·데이터

- 두 모드 모두 4자리 = `kioskCodes[student.id]`(원본 전화 뒤 4자리, PII 마스킹 전 값) 매칭. (기존 로직 재사용)
- 4자리 입력 후 [확인] 클릭 시에만 리스트 표시(현재 동작 유지). 번호를 다시 만지면 리스트 닫힘.
- 복수 매칭(코드 충돌)이면 리스트에 여러 명 표시 → 선택.

## 컴포넌트 구조

`app/kiosk/page.tsx` 한 파일 내에서 모드 상태 머신으로 구성하되, **키패드+확인+학생 리스트(인증 진입)** 부분은 출석/상점이 공유하므로 보조 컴포넌트로 추출해 중복을 없앤다.

- `KioskPage`: `mode`('home' | 'attend' | 'shop') 상태 관리, 모드별 렌더 분기.
- `AuthEntry({ onPick })`(동일 파일 내): 키패드·표시·[확인]·학생 리스트 렌더, 내부 digits/confirmed 상태 관리, 학생 선택 시 `onPick(student)` 호출. 출석 모드에서는 호출 후 내부 입력을 초기화해 키패드로 복귀.
- 홈: 인라인 두 버튼.
- 상점 대시보드: 기존 `done` 단계 렌더를 상점용으로 재사용(배지 제거, 교환 활성).
- 토스트: `KioskPage`의 `toast` 상태 + 화면 하단 고정 표시, `setTimeout`으로 약 2초 후 해제 및 모드 입력 초기화.

## 범위 밖 (YAGNI)

- 실제 인증/백엔드 연동(데모 클라이언트 상태 유지).
- 학생 전용 PIN·스태프 승인 등 추가 신원확인(이번엔 출석과 동일 4자리로 합의).
- 알림톡·영수증 등 부가 기능.
- `내 대시보드` 화면(버튼 비활성 유지).

## 동작 요약

- 출석: 홈 → 출석 체크 → 4자리 → 확인 → 학생 탭 → +30P → 토스트 → (2초) → 키패드 복귀. 교환 불가.
- 상점: 홈 → 포인트 상점 → 4자리 → 확인 → 학생 탭 → 상점 대시보드 → 교환 가능.
