# w키오스크: 카드 탭으로 즉시 체크인 설계

**날짜:** 2026-06-16
**작성자:** Hannah Seo
**상태:** 승인됨

---

## 배경 및 목표

키오스크 출석체크는 번호 입력 후 본인 확인 화면(`ready`)에서 별도의 `✅ 출석 체크인` 버튼을 한 번 더 눌러야 한다. 단일 매칭(대부분의 경우)에서도 `확인` 외에 추가 클릭이 필요해 번거롭다.

**목표:** 별도의 `출석 체크인` 버튼을 없애고, 단일 매칭은 `확인`만으로, 복수 매칭은 이름 선택만으로 곧바로 체크인되어 완료 화면으로 넘어가도록 한다.

---

## 적용 범위

대상 파일: `app/kiosk/page.tsx` (단일 파일)

변경 없는 것: 입력 패드, 8자리 매칭 로직, `select` 화면, `done` 축하 화면, 알림톡 문구, 포인트/칭호 표시.

제거 대상: `ready` 단계(본인 확인 화면)와 그 안의 `출석 체크인` 버튼 — 더 이상 거치지 않음.

---

## 플로우

- **단일 매칭**: 8자리 입력 → `확인` → 바로 `done` (즉시 체크인)
- **복수 매칭**: 8자리 입력 → `확인` → `select`(이름 선택) → 바로 `done` (선택 즉시 체크인)

`select`/`done` 화면에 표시되는 이름으로 본인 확인을 대신한다.

---

## 세부 동작

1. **`handleSearch` 분기**: 매칭 1명이면 `setSelected(found[0])` 후 `handleCheckin()` 호출하여 즉시 `done`.
2. **`select` 클릭**: 이름 버튼 클릭 시 `setSelected(s)` 후 `setStep('ready')` 대신 `handleCheckin()` 호출하여 즉시 `done`.
3. **`ready` 제거**: `ready` 단계 블록, `Step` 타입의 `'ready'`, 해당 화면에서만 쓰이던 `todayClass`/`todaySession` 계산과 `todaySessions` import를 제거한다.
