# 키오스크: 카드 탭으로 즉시 체크인 설계

**날짜:** 2026-06-16
**작성자:** Hannah Seo
**상태:** 승인됨

---

## 배경 및 목표

키오스크 출석체크는 번호 입력 후 본인 확인 화면(`ready`)에서 별도의 `✅ 출석 체크인` 버튼을 한 번 더 눌러야 한다. 원생 입장에서 이 추가 클릭이 번거롭게 느껴진다.

**목표:** 본인 확인 화면의 별도 버튼을 없애고, 본인 정보 카드 자체를 탭하면 바로 체크인되도록 하여 클릭 수를 줄인다.

---

## 적용 범위

대상 파일: `app/kiosk/page.tsx` (단일 파일)

변경 없는 것: 입력 패드, 8자리 매칭 로직, `done` 축하 화면, 알림톡 문구, 포인트/칭호 표시.

---

## 플로우

- **단일 매칭**: 확인 → `ready`(정보 카드 탭) → `done`
- **복수 매칭**: 확인 → `select`(이름 탭) → `ready`(정보 카드 탭) → `done`

`ready`는 항상 최종 확인 단계로 유지한다. 복수 매칭 시에도 `ready`를 거쳐 잘못된 원생이 실수로 체크인되는 것을 방지한다.

---

## 세부 동작

1. **별도 버튼 제거**: `ready` 단계의 `✅ 출석 체크인` 버튼을 삭제한다.
2. **카드 탭 = 체크인**: 본인 정보를 감싸는 카드 영역 전체를 탭 가능한 요소로 만들고, 탭 시 기존 `handleCheckin()`을 호출한다.
3. **탭 어포던스**: 카드 하단에 "탭하여 출석 체크인 →" 같은 안내 문구를 추가하여 탭 가능함을 시각적으로 알린다. `active:scale` 같은 기존 눌림 피드백을 유지한다.
4. **취소 유지**: `취소`(다시 입력) 동작은 작은 텍스트 링크로 그대로 유지한다. 카드 탭 영역과 겹치지 않게 분리한다.

---

## 검증

- 단일 매칭: 8자리 입력 → 확인 → 정보 카드 탭 → `done` 표시 확인.
- 복수 매칭: 입력 → 이름 선택 → `ready` 카드 탭 → `done` 표시 확인.
- `취소` 탭 시 입력 화면으로 초기화되고 체크인되지 않음 확인.
