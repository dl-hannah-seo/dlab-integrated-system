# 키오스크: 카드 탭으로 즉시 체크인 설계

**날짜:** 2026-06-16
**작성자:** Hannah Seo
**상태:** 승인됨

---

## 배경 및 목표

키오스크 출석체크는 번호 입력 후 본인 확인 화면(`ready`)에서 별도의 `✅ 출석 체크인` 버튼을 한 번 더 눌러야 한다. 단일 매칭(대부분의 경우)에서도 `확인` 외에 추가 클릭이 필요해 번거롭다.

**목표:** 단일 매칭일 때 `확인`을 누르면 곧바로 체크인되어 완료 화면으로 넘어가도록 한다. 복수 매칭일 때만 본인 확인 단계를 거친다.

---

## 적용 범위

대상 파일: `app/kiosk/page.tsx` (단일 파일)

변경 없는 것: 입력 패드, 8자리 매칭 로직, `select` 화면, `ready` 화면, `done` 축하 화면, 알림톡 문구, 포인트/칭호 표시.

---

## 플로우

- **단일 매칭**: 8자리 입력 → `확인` → 바로 `done` (`ready` 생략, 즉시 체크인)
- **복수 매칭**: 8자리 입력 → `확인` → `select`(이름 선택) → `ready`(확인 버튼) → `done`

단일 매칭은 `done` 화면에 표시되는 이름으로 본인 확인을 대신한다. 복수 매칭은 동명이인/가족 등 잘못된 원생 체크인을 막기 위해 `ready` 확인 단계를 유지한다.

---

## 세부 동작

1. **`handleSearch` 분기 변경**: 매칭 결과가 1명이면 `setSelected(found[0])` 후 `setStep('ready')` 대신 `handleCheckin()`을 호출하여 즉시 `done`으로 전환한다.
2. **복수 매칭 경로 유지**: 2명 이상이면 기존대로 `setStep('select')` → 이름 선택 시 `setStep('ready')` → `ready`의 `출석 체크인` 버튼 → `done`.
3. **나머지 UI 유지**: `ready` 화면과 그 안의 `출석 체크인` 버튼은 복수 매칭용으로 그대로 둔다.

---

## 검증

- 단일 매칭: 8자리 입력 → `확인` → 추가 클릭 없이 바로 `done` 표시, 시간/포인트 정상 확인.
- 복수 매칭: 입력 → `확인` → `select`에서 이름 선택 → `ready` → `출석 체크인` → `done` 확인.
- 불일치 번호: `확인` 시 오류 문구 표시, 체크인되지 않음 확인.
